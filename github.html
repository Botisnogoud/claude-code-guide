🐙 Integration

GitHub Integration

Connect Claude Code to GitHub with OAuth, Personal Access Tokens, or GitHub Apps — enabling PR creation, issue management, and CI/CD automation.

Method 1

OAuth Authentication (Easiest)

The simplest way — authenticate Claude Code directly with your GitHub account in seconds.

1
Run the Setup Command
Inside any Claude Code session, run:
bash
claude setup-token
2
Authorize in Browser
A browser window opens. Log in to GitHub and click "Authorize Anthropic Claude Code". This grants Claude the permissions listed below.
3
Test the Connection
Back in your terminal, try asking Claude to list your repos or create a PR.
bash
claude "create a pull request for my current branch with a description of changes"
💡
Who can use OAuth? Available on Claude Pro, Max, Team, and Enterprise plans. Free plan users should use PATs instead.
Method 2

Personal Access Token (PAT)

For fine-grained control or when OAuth isn't available — create a token with only the permissions Claude needs.

1
Go to GitHub Token Settings
Navigate to GitHub → Settings → Developer Settings → Personal Access Tokens → Fine-grained tokens
2
Set Minimal Permissions
Only grant what Claude needs. Use the principle of least privilege:
PermissionAccess LevelWhy Needed
actionsReadView CI/CD status
contentsWritePush commits, read files
issuesWriteCreate & comment on issues
metadataReadView repo info (required)
pull_requestsWriteOpen & manage PRs
3
Set an Expiry Date
Always set an expiration — 30 or 90 days is recommended. Rotate tokens regularly.
4
Add Token to Claude Code
Set the token as an environment variable or pass it during setup:
bash
# Add to your shell profile (~/.bashrc or ~/.zshrc)
export GITHUB_TOKEN="github_pat_xxxxxxxxxxxx"

# Or configure directly
claude mcp add github --token github_pat_xxxxxxxxxxxx
⚠️
Never commit tokens to git! Add .env and any file containing your token to .gitignore. Use GitHub Secrets for CI/CD workflows.
CI/CD

GitHub Actions Integration

Run Claude Code in your CI/CD pipeline to automatically review PRs, generate documentation, or fix failing tests.

yaml — .github/workflows/claude-review.yml
name: Claude Code Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Run Claude Code Review
        uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          github_token: ${{ secrets.GITHUB_TOKEN }}
          task: |
            Review this pull request for:
            1. Code quality issues
            2. Security vulnerabilities
            3. Missing tests
            4. Breaking changes
            Post a detailed review comment on the PR.
✅
Store ANTHROPIC_API_KEY in GitHub → Settings → Secrets and Variables → Actions. Never hardcode keys in workflow files.

More Workflow Templates

🔧 Auto-Fix Lint Errors
Triggered on PR — Claude runs ESLint, reads error output, then edits the failing files to fix all lint errors automatically.
yaml — .github/workflows/claude-lint-fix.yml
name: Claude Auto-Fix Lint
on:
  pull_request:
    types: [opened, synchronize]
jobs:
  lint-fix:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          github_token: ${{ secrets.GITHUB_TOKEN }}
          task: |
            Run `npm run lint 2>&1` and read the output.
            Fix every lint error in the affected files.
            Commit the fixes with message: "fix: auto-fix lint errors via Claude"
📚 Auto-Generate API Docs
Triggered when API route files change — Claude reads the changed routes and updates the API reference documentation.
yaml — .github/workflows/claude-docs.yml
name: Claude API Docs
on:
  push:
    paths: ['src/api/**', 'src/routes/**']
    branches: [main]
jobs:
  update-docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          github_token: ${{ secrets.GITHUB_TOKEN }}
          task: |
            Read all changed API route files.
            Update docs/api-reference.md with accurate endpoint documentation
            including: method, path, request body schema, response schema, example curl.
            Commit with: "docs: update API reference for changed routes"
🔒 Security Scan on Push
Runs a security audit agent on every push to main — posts findings as a GitHub commit comment and fails if critical issues found.
yaml — .github/workflows/claude-security.yml
name: Claude Security Scan
on:
  push:
    branches: [main]
jobs:
  security:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          github_token: ${{ secrets.GITHUB_TOKEN }}
          task: |
            Use the security-auditor agent to scan all changed files.
            Check for: SQL injection, XSS, hardcoded secrets, insecure deps.
            If any Critical findings exist, exit with code 1 to fail the build.
            Post all findings as a commit comment.
Method 3

GitHub App (Team/Enterprise)

For organizations — create a GitHub App for centralized access control across all team repositories.

1
Create GitHub App
Go to GitHub Organization Settings → Developer Settings → GitHub Apps → New GitHub App. Use the Anthropic manifest for one-click setup:
json — app-manifest.json
{
  "name": "Claude Code Bot",
  "description": "AI-powered code assistant",
  "url": "https://claude.ai",
  "webhook_active": false,
  "public": false,
  "default_permissions": {
    "actions": "read",
    "contents": "write",
    "issues": "write",
    "pull_requests": "write",
    "metadata": "read"
  }
}
2
Install on Repositories
After creating, install the app on specific repositories or all org repos. Fine-grained access per repo.
3
Configure in Claude Code
Add the App ID and private key to your Claude Code config or CI secrets.
bash
export GITHUB_APP_ID="12345"
export GITHUB_APP_PRIVATE_KEY_PATH="/path/to/private-key.pem"
🧒 ELI5 — What's the difference between OAuth, PAT, and GitHub App? ▼

Imagine you want Claude to enter your school (GitHub) to help you:

🪪 OAuth is like using your school ID card. Quick and easy — just swipe and you're in. Best for personal use.

🔑 PAT (Personal Access Token) is like a special key you cut yourself. You decide which doors it opens and which it doesn't. Best when you want tight control.

🏫 GitHub App is like giving Claude an official visitor badge for the whole school building. Admins control it, it works for everyone, and you can revoke it centrally. Best for teams.

Security

GitHub Security Checklist

  • Use fine-grained tokens (not classic tokens) with minimal scopes
  • Set expiry dates on all tokens — rotate every 30–90 days
  • Never commit tokens, keys, or credentials to any git repository
  • Use GitHub Actions Secrets for CI/CD credentials
  • Review OAuth app permissions quarterly — revoke unused apps
  • Enable two-factor authentication on your GitHub account
  • Audit Claude's GitHub actions via repository event logs
  • For teams, use GitHub App (not PATs) for centralized control
