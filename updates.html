MEMORY.md Starter Template
markdown — ~/.claude/projects/<project>/memory/MEMORY.md
# MEMORY.md
Persistent memory for Claude Code sessions in <project>

## Architecture
- Stack: Next.js 14 (App Router) + PostgreSQL + Prisma ORM
- Auth: NextAuth.js with Google + GitHub providers
- Hosting: Vercel (frontend) + Railway (DB)

## Key File Paths
- DB schema: prisma/schema.prisma
- Auth config: src/lib/auth.ts
- API routes: src/app/api/
- Shared types: src/types/index.ts

## Gotchas
- NEVER use useEffect for data fetching — use React Query
- DB migrations require: npx prisma migrate dev --name <name>
- Environment: .env.local (gitignored), .env.example (committed)

## User Preferences
- No auto-commit under any circumstances
- Always run: npm run lint && npm run typecheck before finishing
- Prefer named exports over default exports

Parallel Agents Pattern
markdown — .claude/agents/parallel-qa.md
---
name: parallel-qa
description: Runs test scan, security scan, and doc check simultaneously
tools:
  - Agent
  - Read
  - Glob
isolation: worktree
---

# Parallel QA Agent

Run all three checks simultaneously. Do NOT run sequentially.

## Step 1 — Fan Out (all three in one response)
Invoke these three agents in parallel:
1. Agent: test-runner — run full test suite, report failures
2. Agent: security-scanner — check for hardcoded secrets, SQL injection, XSS
3. Agent: doc-checker — verify all exported functions have JSDoc comments

## Step 2 — Merge Results
Wait for all three to complete, then produce a single report:
- PASS / FAIL status per check
- Actionable list of issues (file:line for each)
- Overall verdict: READY TO MERGE or NEEDS FIXES

PostToolUse Hook — Auto Test Runner
json — .claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit|MultiEdit",
        "hooks": [
          {
            "type": "command",
            "command": "npm run lint --silent && npm run typecheck --silent",
            "timeout": 30000,
            "onFailure": "notify"
          }
        ]
      },
      {
        "matcher": "Write",
        "filePattern": "**/*.test.ts",
        "hooks": [
          {
            "type": "command",
            "command": "npx jest --testPathPattern="$CLAUDE_TOOL_FILE" --passWithNoTests",
            "timeout": 60000,
            "onFailure": "ask_claude_to_fix"
          }
        ]
      }
    ]
  }
}
4
Minimal Agent File — Least-Privilege Pattern
markdown — .claude/agents/code-reviewer.md
---
name: code-reviewer
description: Reviews code for security, performance, and style issues. Read-only — never writes files.
tools:
  - Read
  - Glob
  - Grep
model: claude-sonnet-4-6
isolation: none
---

# Code Reviewer

You are a senior code reviewer. Your job is to find issues, not fix them.
You have READ-ONLY access. Never suggest using Write or Edit.

## Review Checklist
1. Security: hardcoded secrets, SQL injection, unvalidated inputs, CORS misconfig
2. Performance: N+1 queries, missing indexes, synchronous I/O in hot paths
3. Types: any casts, missing null checks, uncaught promise rejections
4. Style: naming consistency, dead code, missing error handling

## Output Format
For each issue found:
- File: path/to/file.ts:lineNumber
- Severity: CRITICAL / HIGH / MEDIUM / LOW
- Issue: one-sentence description
- Fix: concrete suggestion (do not implement it)

settings.json — Power User Config
json — ~/.claude/settings.json
{
  "model": "claude-sonnet-4-6",
  "autoCompact": true,
  "statusLine": {
    "show": ["model", "tokens", "cost"]
  },
  "env": {
    "ANTHROPIC_SMALL_FAST_MODEL": "claude-haiku-4-5-20251001"
  },
  "agents": {
    "parallel": true,
    "maxConcurrent": 3
  },
  "permissions": {
    "deny": [
      "Bash(rm -rf*)",
      "Bash(git push --force*)",
      "Bash(sudo rm*)",
      "Bash(DROP TABLE*)",
      "Bash(curl * | bash)"
    ]
  },
  "hooks": {
    "Stop": [
      {
        "type": "command",
        "command": "echo "Session ended: $(date)" >> ~/claude-sessions.log"
      }
    ]
  }
}
