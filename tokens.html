💰 Cost Efficiency

Token Optimization
Best Practices

Tokens are Claude Code's currency. Every file read, every command, every response costs tokens. Learn to do more with less — without sacrificing quality.

Basics

Understanding Token Usage

Everything Claude processes costs tokens. Understanding what's expensive helps you optimize.

📈
High Token Cost
Large files read in full, long conversation history, many MCP servers loaded, complex system prompts.
📊
Medium Token Cost
Normal code edits, single skill loaded, GitHub tool calls, short bash commands with output.
📉
Low Token Cost
Targeted file reads, compact summaries, cached prompts, no unnecessary MCP servers.
bash — monitor usage in real time
# Check current session cost
/cost

# Output example:
# Tokens used: 45,230 input / 8,940 output
# Estimated cost: $0.48 (based on current model pricing)
# Context: 45% full
🧒 ELI5 — What's a token? ▼

Imagine each word you write costs 1 penny to send to Claude. If you send a whole book, that's very expensive! If you send just the chapter Claude needs, that's cheap.

A "token" is roughly one word (or part of a word). Claude can only hold a certain number of tokens in memory at once — like a brain that can only think about a certain number of things. When you use too many tokens, older things get forgotten. So we want to use tokens wisely — only sending Claude what it actually needs!

Strategies

Top Optimization Strategies

1
Use /compact — Your Best Friend
When a session gets long, /compact replaces the full conversation history with a short summary — saving thousands of tokens while keeping the key context.
bash
# After a long debugging session, before starting a new task:
/compact

# Or compact with a custom summary focus:
/compact "we fixed the auth bug, now moving to the payment module"
2
Start Fresh Sessions for Unrelated Tasks
Don't mix unrelated tasks in one chat. A debugging session about auth has no useful context for a UI task. Use /clear or start a new session.
bash
# Start fresh for a new unrelated task
/clear

# Or just open a new terminal session
claude "now help me with the dashboard component"
3
Use Targeted File Reads
Tell Claude exactly which files and lines to read instead of letting it explore freely. Guide it to the relevant code.
bash
# Expensive — Claude reads many files to explore:
claude "fix the login bug"

# Efficient — guide Claude to the right files:
claude "fix the login bug in src/auth/login.ts around line 45
        where the JWT validation happens"
4
Disable Unused MCP Servers
Each active MCP server adds hundreds of tool definitions to the system prompt. Disable any server you're not actively using.
bash
# See which servers are active and their token cost:
/mcp

# Disable a server you don't need right now:
/mcp disable stripe

# Or in settings.json, set "disabled": true for servers not in use
5
Keep CLAUDE.md Concise
CLAUDE.md loads every session. Every extra line = tokens wasted on every single interaction. Audit it regularly.
✅ Concise
Use TypeScript strict mode. No `any`.
❌ Verbose
When writing TypeScript code, always make sure to enable TypeScript's strict mode by setting "strict": true in the tsconfig.json file, and avoid using the `any` type as it defeats the purpose of using TypeScript...
6
Use Skills for Progressive Disclosure
Instead of including all instructions in CLAUDE.md, use Skills — they load only when needed, recovering up to 82% of tokens per session.
✅
Skills metadata (~200 tokens) loads at start. Full skill content (~5,000 tokens) loads only when triggered. On a typical session, this saves ~15,000 tokens.
7
Use Cheaper Models for Simple Tasks
Route simple tasks to faster, cheaper models. Use Haiku for boilerplate, Sonnet for standard tasks, Opus only for complex architectural reasoning.
json — settings.json
{
  "env": {
    "ANTHROPIC_SMALL_FAST_MODEL": "claude-haiku-4-5-20251001"
  }
}
Automatic

Automatic Optimizations

Claude Code handles these automatically — but knowing about them helps you design workflows that benefit from them.

⚡
Prompt Caching
Repeated system prompts (CLAUDE.md, skills metadata) are cached. Subsequent sessions pay 90% less for the same content.
🗜️
Auto-Compaction
When context reaches 95% full, Claude automatically summarizes older messages to stay under the limit.
🧠
OpusPlan Routing
In plan mode, Opus handles the thinking, Sonnet handles implementation. You get Opus quality at Sonnet cost for most work.
Prompt Engineering

10 Prompting Techniques That Reduce Token Use

1
Specify exact file paths and line numbers
Instead of "fix the login bug" → "fix the bug in src/auth/login.ts:45 where JWT validation fails on expired tokens." Saves 1k–5k tokens of exploration.
2
Tell Claude what NOT to read
"Fix the header component. Don't read any files except src/components/Header.tsx and its imports." Prevents unnecessary file exploration.
3
Pre-filter results before asking Claude
Run grep -n "pattern" file.ts yourself, then include the line numbers in your prompt. Claude reads only those lines, not the whole file.
4
Use code snippets instead of file paths
For small changes, paste the relevant 20 lines directly into the prompt instead of asking Claude to read a 500-line file to find them.
5
Set scope boundaries explicitly
"Only modify files in src/payments/. Do not touch src/auth/ or test files." Prevents Claude from making unrequested broad changes.
6
One task per session
Start a fresh session (/clear) for each unrelated task. Mixing tasks causes context bloat — old context is still paid for even if irrelevant.
7
Use /compact with a focus hint
/compact "we fixed auth, now working on payment UI" — the hint ensures the summary keeps the right context and discards the rest.
8
Ask for summaries, not full rewrites
"Summarize what changes are needed in a numbered list. Don't implement yet." Then you review before Claude writes code — avoiding wasted implementation tokens.
9
Disable verbose output modes
Add to CLAUDE.md: "Be concise. Skip obvious explanations. No preamble like 'Of course!' or 'Great question!'. Just do the task." Cuts response tokens 20–30%.
10
Use Haiku for drafting, Sonnet for final
Generate a first draft with Haiku (cheap), review it, then ask Sonnet to refine only the problematic parts. Saves 60–80% on draft generation cost.
