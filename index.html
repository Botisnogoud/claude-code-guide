Installation

Installing Claude Code

Claude Code is a CLI that runs in your terminal and connects to Anthropic's AI models.

1
Install via NPM
Requires Node.js 18+.
bash
npm install -g @anthropic-ai/claude-code
2
Authenticate
Run claude — it opens a browser OAuth flow. No API key needed for Pro/Max/Team/Enterprise plans.
bash
claude
# Opens browser → Log in → Returns to terminal
3
Set Default Model
Lock in the latest and best coding model explicitly.
json — ~/.claude/settings.json
{ "model": "claude-sonnet-4-6" }
4
Init Your Project
Claude reads your codebase and auto-generates a CLAUDE.md tailored to your stack.
bash
cd your-project && claude /init
5
Enable Auto-Compact
Sessions auto-compact at 85% context — no more failed sessions at the limit.
json — ~/.claude/settings.json
{ "model": "claude-sonnet-4-6", "autoCompact": true }
6
Set Up Status Line
See model, token count, and cost after every tool call — never get surprised by a runaway session.
json — ~/.claude/settings.json
{
  "model": "claude-sonnet-4-6",
  "autoCompact": true,
  "statusLine": { "show": ["model", "tokens", "cost"] }
}
7
Install VS Code Extension
Search Claude Code in the VS Code Marketplace — inline diffs, sidebar chat, right-click file pinning.
bash
code --install-extension claudeai.claude-code
🧒 ELI5 — What is Claude Code? ▼

Imagine you have a super-smart robot friend who lives inside your computer's text box (the terminal). You can type messages to it like "help me fix this bug" or "write tests for my code", and it actually goes and does those things!

It's like having an expert programmer sitting next to you who never gets tired, knows every coding language, and can read, edit, and run your code files — all just from a simple conversation.

Claude Code is that robot friend. You install it once, log in, and then just chat with it inside any project folder. It understands your whole codebase and helps you build things faster.

File System

The .claude Folder Structure

All Claude Code configuration lives in the .claude directory — both globally in your home folder and locally per project.

📁 Global (applies to all projects)

~/.claude/ ├── settings.json # Your global preferences & permissions ├── commands/ # Custom slash commands (reusable prompts) │ └── commit.md ├── skills/ # Global skills (auto-detected by Claude) │ └── WritingTests.md ├── agents/ # Global sub-agent definitions │ └── code-reviewer.md └── worktrees/ # Isolated git worktrees (auto-managed)

📁 Per-Project (inside your repo)

your-project/ ├── CLAUDE.md # Project instructions (committed to git) ├── .claude/ │ ├── settings.json # Team-shared settings (committed) │ ├── settings.local.json # Your personal settings (git-ignored) │ ├── commands/ # Project-specific slash commands │ ├── skills/ # Project-specific skills │ └── agents/ # Project-specific sub-agents ├── src/ └── package.json
💡
Priority Order: Enterprise policies → CLI flags → .claude/settings.local.json → .claude/settings.json → ~/.claude/settings.json. Local always wins over global.
🧒 ELI5 — Why two .claude folders? ▼

Think of your house (your computer) and your school (your project folder). You have rules that apply everywhere — like "be polite" — those go in your home. But your school also has its own rules — like "no phones in class" — those stay at school.

The global ~/.claude is your home rules. The project .claude/ is your school rules. Claude always follows both, but school rules win when they conflict.

Setting Up Slack Integration

1
Install the Claude for Slack App
Go to claude.ai → Integrations → Slack and click "Add to Slack". Authorize for your workspace.
2
Invite Claude to a Channel
In Slack, type /invite @Claude in any channel where you want coding help.
3
Start a Coding Session
Mention @Claude in any message. It creates a thread and assists with your task directly in Slack.
slack
@Claude help me write a Python function to parse JSON from an API response

Setting Up Google Drive Integration

1
Connect in Settings
Go to claude.ai → Settings → Integrations → Google Drive and click Connect.
2
Authorize Google OAuth
Grant read-only access to your Drive. Claude can view but not modify files unless you explicitly share them.
3
Reference Docs in Prompts
Share a Google Docs link in your prompt and Claude will read and use its content as context.
🧒 ELI5 — What are integrations? ▼

Imagine Claude is your helper who can only work inside one room. Integrations are like giving Claude a pass to walk into other rooms too — like the Slack chat room, or the Google Docs room.

Without integrations, Claude can only see your code files. With integrations, Claude can also peek at your Slack messages, read your Google Docs, and even create GitHub pull requests for you!
