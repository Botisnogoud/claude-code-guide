CommandDescription

⌨️

Context Files

CLAUDE.md Sections

What to put in each section of your CLAUDE.md for maximum effectiveness.

📌
Project Overview
1-2 paragraph description: what the project does, who uses it, and why it exists. Context for every session.
🛠️
Tech Stack
Language, framework, major libraries, Node/Python version, package manager. No guessing from package.json.
▶️
Commands
Exact run, test, build, and deploy commands. Claude uses these verbatim — accuracy is critical.
🗂️
Architecture
Annotated folder tree. Key files and their roles. Data flow for the main use case.
✏️
Code Style
Naming conventions, import order, formatting rules, patterns to follow and patterns to avoid.
⛔
Never Do
Explicit list of things Claude must never do: commit without permission, use specific deprecated APIs, etc.
markdown — CLAUDE.md skeleton
# Project Name

## Overview
[1-2 paragraphs: what, who, why]

## Tech Stack
- Language: TypeScript 5.4
- Framework: Express 4.18
- Database: PostgreSQL 15 (via Prisma ORM)
- Test runner: Jest 29
- Package manager: pnpm

## Commands
- Run: `pnpm dev`
- Test: `pnpm test`
- Build: `pnpm build`
- Migrate DB: `pnpm db:migrate`

## Architecture
[annotated folder tree]

## Code Style
[naming, patterns, conventions]

## Never Do
- Never commit without explicit user instruction
- Never use `any` type in TypeScript
- Never run `pnpm db:migrate` without showing migration first
Automation

Hook Events

Hooks let you run shell commands before or after Claude's tool calls — for logging, validation, notifications, and guardrails.

EventFiresCan block?Common use
PreToolUseBefore any tool call executesYes (exit code ≠ 0)Validate args, log audit trail, block dangerous commands
PostToolUseAfter a tool call completesNoLog results, trigger notifications, update dashboards
NotificationWhen Claude has a message for the userNoSend Slack/Teams alert for long-running tasks
StopWhen Claude finishes a responseNoCleanup, post-session reporting, metrics
SubagentStopWhen a sub-agent completesNoLog sub-agent output, chain workflows

Hook configuration examples

json — settings.json hooks
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "bash /home/user/.claude/hooks/audit-bash.sh"
          }
        ]
      }
    ],
    "Notification": [
      {
        "matcher": ".*",
        "hooks": [
          {
            "type": "command",
            "command": "notify-send 'Claude Code' '$CLAUDE_NOTIFICATION'"
          }
        ]
      }
    ],
    "Stop": [
      {
        "matcher": ".*",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"Session ended at $(date)\" >> ~/.claude/session.log"
          }
        ]
      }
    ]
  }
}
🔴
Blocking with PreToolUse: Your hook script must exit with code 0 to allow the tool call. Any non-zero exit code blocks it. Use exit 1 with a message on stdout — Claude will see your message and explain to the user.
Security

Permission Patterns

Allow/deny syntax for permissions.allow and permissions.deny in settings.json.

json — common permission patterns
{
  "permissions": {
    "allow": [
      "Bash(git *)",              // All git subcommands
      "Bash(npm test*)",          // npm test and variants
      "Bash(npm run lint*)",      // npm run lint
      "Bash(npx tsc*)",           // TypeScript compiler
      "Read",                     // All file reads (always on)
      "Write(src/**)",            // Write only inside src/
      "Edit(src/**)"              // Edit only inside src/
    ],
    "deny": [
      "Bash(rm -rf *)",           // Block recursive delete
      "Bash(git push --force*)",  // Block force push
      "Bash(git reset --hard*)",  // Block hard reset
      "Bash(curl *)",             // Block arbitrary HTTP
      "Bash(wget *)",             // Block wget downloads
      "Write(.env*)",             // Block writing env files
      "Write(*.key)",             // Block writing key files
      "Write(secrets/**)"         // Block secrets directory
    ]
  }
}
Pattern syntaxMatches
Bash(git *)Any bash command starting with git
Bash(npm test*)npm test, npm test:watch, npm test -- --coverage
Write(src/**)Write calls to any path under src/
Write(*.md)Write calls to any .md file
ReadAll Read tool calls (no path filter)
WebFetch(https://docs.example.com*)Fetch calls to that domain only

Task typeTypical tokensSonnet cost (approx.)Notes
Quick question / small edit1K–5K<$0.01Single file read + small write
Write a test suite (1 file)5K–15K$0.01–$0.04Read source + write tests
Code review (small PR)10K–30K$0.03–$0.09Reads diff + multiple files
Scaffold new feature20K–60K$0.06–$0.18Reads patterns + writes files
Full codebase analysis50K–200K$0.15–$0.60Glob + read many files

Large refactor (30+ files)200K–600K$0.60–$1.80Reads all files + writes all files
⚠️
These are estimates. Actual costs vary based on codebase size, number of tool calls, and how much conversation history accumulates. Use /cost in-session to see real numbers. Use /compact to reduce context when costs spike.
💰
Reduce costs
Use /compact to compress long sessions. Use Haiku for simple sub-tasks. Avoid re-reading large files repeatedly — reference previous reads.
📊
Track spending
Check /cost mid-session. Use Anthropic Console usage dashboard for daily/monthly totals per API key.
⚡
Max value
Front-load context in CLAUDE.md so Claude doesn't re-explore. Targeted prompts with specific file paths cost far less than "look around the codebase".
